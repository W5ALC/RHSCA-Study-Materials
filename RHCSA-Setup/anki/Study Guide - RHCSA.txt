>	Tells shell to connect stdout of a process to a specified file. If the file does not exist, it will be created. If it does exist, it's contents will be over-written.

What are the general characteristics of RAID 0?	<div>- Splits data evenly across stripe sets/striped volumes on 2 or more disks.</div><div>- No parity information is stored.</div><div>- There is no redundancy.</div><div>- There is no fault tolerance.</div><div>- Failure of one drive will cause the entire array to fail.</div><div>- RAID 0 is used to increase speed performance.</div>
What is striping in RAID 0?	<div>- Striping spreads data across 2 or more physical drives.</div><div>- The data is divided into blocks, and those blocks are spread across stripe sets/striped volumes on 2 or more disks.</div><div>- The disks can be hard drives, SSDs, etc.</div><div>- Multiple stripes increase read/write throughput, leading to better read/write performance.</div><div>- No fault tolerance or redundancy makes striping best suited for non-critical data sets.</div>
Discuss RAID parity information.	<div>- Uses a parity bit or check bit.</div><div>- Parity bits are the simplest form of error detecting code.</div><div>- A bit is added to a string of binary code to ensure that the total number of 1-bits in the string is even or odd.</div><div>- There are 2 variants of parity bits:</div><div>1.) even parity bits.</div><div>2.) odd parity bits.</div><div>- Error detection: If the number of bits, including the parity bit, are transmitted incorrectly, this means that the parity bit will be incorrect, indicating a parity error in the transmission.</div>
